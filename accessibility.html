Success Criterion 1.4.1: Use of Color (Level A)

Overview

Colour is used as the only visual means of distinguishing {{WHAT}}.

Details

{{COLOUR}}.

Impact

  • Some users may have problems distinguishing between colours.
  • Some older users may not be able to see colour well.
  • Users of screen readers may be unable to access colour-dependent information.

Recommendation

Eliminate colour dependency by including a heading such as "alert", "attention", "error" or "important", for example:

    Alert: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

The heading should be represented by a header element in the underlying HTML.

Source

Success Criterion 1.4.1: Use of Color from the Web Content Accessibility Guidelines.

Success Criterion 1.4.8: Visual Presentation (Level AAA)

Overview

Text is justified or centred aligned.

Details

The {{WHAT}} piece of text is centred aligned.

Impact

  • Centred aligned text is more difficult for most people to read, when there are multiple lines of text. It is harder to find the beginning of the next line of text and keep a smooth reading flow for comprehension.
  • On translated web pages it is also harder for people to read vertically from top to bottom or horizontally from right to left in some Asian and Middle Eastern languages. 
  • Users of screen magnifiers may miss some centred text.

Recommendation

Align text to the left.

Source

Success Criterion 1.4.8: Visual Presentation from the Web Content Accessibility Guidelines.

Success Criterion 2.4.4: Link Purpose (In Context) (Level A)

Overview

The purpose of the {{WHAT1}} link cannot be efficiently determined.

Details

The context of the {{WHAT1}} link is {{WHAT2}}. The context of a link helps the user understand the link's purpose so they can decide whether they want to follow the link. However,  associating a link with its context isn't efficient when there's a considerable amount of white space in between. The user should be able to identify the purpose of a link without moving focus from the link itself. In other words, the user can arrive on this link and find out more about it without losing his place.

Impact

  • Users of screen magnifiers may not be able to determine the purpose of the link if the link's context is off the screen.
  • Users with a visual impairment may find it difficult to associate the link and its context as they move across the white space in between.

Recommendation

  • Put the description of the link in the same sentence, paragraph, list item, or table cell as the link, or in the table header cell for a link in a data table, because these are directly associated with the link itself.
  • Make sure the purpose of the link can be understood in isolation.

Source

Success Criterion 2.4.4: Link Purpose from the Web Content Accessibility Guidelines.

Success Criterion 3.2.2: On Input (Level A)

Overview

The {{WHAT}} form submits automatically when {{WHAT}}.

Details

This form has been designed so that it submits automatically when the user has filled in all the fields. It is better to rely on the standard form behaviour of the submit button and enter key.

Impact

  • Some users may be confused as it isn't clear that automatic form submission will happen.
  • Some users with hand tremors may not have a chance to correct data if they have entered it incorrectly.

Recommendation

Avoid the automatic form submission and, instead, provide a submit button.

Success Criterion 3.2.4: Consistent Identification (Level AA)

Overview

The {{WHAT1}} and {{WHAT2}} cannot be identified consistently.

Details

The {{WHAT1}} and {{WHAT2}} have identical functions. However, they have different labels on different pages.

Impact

  • Some users of screen readers, who learn functionality on one page, may not find the same functionality on other pages. For that reason, the pages may be more difficult to use.
  • Some users with cognitive limitations may be confused and the cognitive load may increase as a result.

Recommendation
